# 🏠 家庭事项管理系统 - 完整测试指南 (Testing Guide)

本项目是一个集成 AI 解析、循环任务引擎、持久化登录以及数据导出功能的家庭事务协作平台。本指南旨在详细列出所有最新功能测试点及应用场景，确保系统运行稳健。

---

## 🔐 第一部分：安全与访问控制 (Authentication)

### 1.1 基础登录测试
*   **场景**: 用户首次访问网站。
*   **测试点**:
    - [ ] 页面是否显示“家庭系统登录”标题？
    - [ ] 输入【错误密码】是否显示“🚫 密码错误”提示？
    - [ ] 输入【正确密码 (790228)】是否能瞬时跳转至主界面？

### 1.2 登录持久化 (Persistent Login)
*   **场景**: 用户已登录，希望刷新网页不掉线。
*   **测试点**:
    - [ ] 登录成功后，手动刷新浏览器页面，是否直接显示内容（无需再输密码）？
    - [ ] 关闭浏览器标签页，重新打开网址，是否依然保持登录状态？
    - [ ] 在侧边栏点击“🔴 退出登录”，刷新网页，是否要求重新输入密码？

---

## 🧠 第二部分：AI 智能任务解析 (AI Parsing)

### 2.1 自然语言日期提取
*   **场景**: 用户在侧边栏“新增事项”框输入一段话。
*   **测试案例**:
    - [ ] 输入 `今晚8点去超市买牛奶` -> 识别为今日日期，时间 20:00。
    - [ ] 输入 `明天下午3点接孩子` -> 识别为明日日期。
    - [ ] 输入 `下周五看牙医` -> 识别为下周对应的具体日期。
    - [ ] 输入 `4月3日去旅游` -> 识别为 2026-04-03（以后待办）。
    - [ ] **标题清理验证**: 检查任务标题中是否已自动剥离“明天”、“下午3点”等冗余词汇，仅保留核心任务内容。

### 2.2 循环意图提取
*   **场景**: 添加长期重复性事务。
*   **测试案例**:
    - [ ] 输入 `每周二购买 Game Night 零食` -> 提取循环模式：`Tuesday`。
    - [ ] 输入 `每天早上8点准时吃药` -> 提取循环模式：`Everyday`。
    - [ ] 输入 `每个周末打扫卫生` -> 提取循环模式：`Weekend`。

---

## 📝 第三部分：交互式任务管理 (Task Interaction)

### 3.1 任务新增与反馈 (New!)
*   **场景**: 用户在侧边栏点击“立即添加”。
*   **测试点**:
    - [ ] **内容解析确认**: 点击后是否弹出“📋 事项添加结果”对话框？
    - [ ] **数据核对**: 对话框中显示的“内容”、“预计执行时间”和“循环模式”是否准确反映了 AI 的理解？
    - [ ] **自动复位**: 关闭弹窗（点击确定）后，侧边栏的输入框是否已自动变为空白？
    - [ ] **立即显示**: 任务是否已实时出现在右侧对应的列表区域？

### 3.2 列表自动分类
*   **区域验证**:
    - [ ] **⚡ 今日急需处理**: 截止日期为今天及之前的任务。
    - [ ] **🌙 明日处理事项**: 截止日期正好为明天的任务。
    - [ ] **🗓️ 本周剩余任务**: 截止日期在明天之后到本周日之前的任务。
    - [ ] **⏳ 以后待办**: 所有本周以后（下周一及以后）的任务。

### 3.3 任务操作 (Edit & Delete)
*   **测试点**:
    - [ ] **行内编辑**: 点击任务右侧的 【✏️ 铅笔】图标，任务是否变为输入框？修改文字后点击【💾 保存】，列表是否即时更新？
    - [ ] **完成确认**: 点击任务左侧【复选框】，任务文字是否变灰带删除线？切换至“✅ 已完成事项”标签页验证其是否存在。
    - [ ] **物理删除**: 点击任务右侧【🗑️ 垃圾桶】图标，任务是否从界面中彻底消失？

## 🔄 第四部分：循环任务引擎 (Recurring Engine)

### 4.1 独立管理测试
*   **测试点**:
    - [ ] 进入“🔄 循环事项”标签页，是否列出了所有带循环标签的任务？
    - [ ] 在此页面删除一个循环任务，主页面的“影子提醒”是否同步消失？

### 4.2 智能影子注入 (Shadow Injection)
*   **场景**: 循环任务在每日清单中的提醒功能。
*   **测试点**:
    - [ ] 若有一个任务是“每周二...”，且今天正好是周二：
        - 该任务是否出现在“今日急需处理”中？
        - 图标是否为旋转的 🔄？是否没有复选框（仅作为提醒）？
    - [ ] 检查“明日处理事项”：如果明天符合循环条件，明日区域是否也出现了该任务的提醒？

---

## 📊 第五部分：数据导出与存档 (Data Export)

### 5.1 任务报表导出 (New!)
*   **场景**: 用户希望离线保存所有任务。
*   **测试点**:
    - [ ] 点击主标题右侧的 **📥 下载任务报表 (TXT)** 按钮。
    - [ ] **文件命名**: 检查下载的文件名是否包含当前日期和具体时间（如 0311_1000）。
    - [ ] **格式核对**: 打开 TXT 文件，检查是否呈现为整齐的表格？是否包含了“截止时间”、“任务内容”、“循环模式”和“完成状态”四列？
    - [ ] **数据覆盖**: 检查报表中是否包含了当前活跃任务、循环任务以及所有已完成的任务？

---

## 📱 第六部分：异常与响应式测试

### 6.1 异常处理
*   **测试点**:
    - [ ] API Key 失效或网络波动时，添加任务是否通过对话框给出明确的错误原因而非网页崩溃？
    - [ ] 多次快速点击“立即添加”，系统是否能稳健处理回调，不会导致输入框状态重叠报错？

### 6.2 移动端匹配
*   **场景**: 使用手机/平板访问。
*   **测试点**:
    - [ ] 侧边栏在手机上是否能自动收起？
    - [ ] 任务列表在窄屏下是否由于 layout="wide" 而能自适应，且文字不会溢出？
