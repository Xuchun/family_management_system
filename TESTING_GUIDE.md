# 🏠 家庭事项管理系统 - 完整测试指南 (Testing Guide)

本项目是一个集成 AI 解析、循环任务引擎和持久化登录的家庭事务协作平台。本指南旨在详细列出所有功能测试点及应用场景，确保系统运行稳健。

---

## 🔐 第一部分：安全与访问控制 (Authentication)

### 1.1 基础登录测试
*   **场景**: 用户首次访问网站。
*   **测试点**:
    - [ ] 页面是否显示“家庭系统登录”标题？
    - [ ] 输入【错误密码】是否显示“🚫 密码错误”提示？
    - [ ] 输入【正确密码 (790228)】是否能瞬时跳转至主界面？

### 1.2 登录持久化 (Persistent Login)
*   **场景**: 用户已登录，希望刷新网页不掉线。
*   **测试点**:
    - [ ] 登录成功后，手动刷新浏览器页面，是否直接显示内容（无需再输密码）？
    - [ ] 关闭浏览器标签页，重新打开网址，是否依然保持登录状态？
    - [ ] 在侧边栏点击“🔴 退出登录”，刷新网页，是否要求重新输入密码？

---

## 🧠 第二部分：AI 智能任务解析 (AI Parsing)

### 2.1 自然语言日期提取
*   **场景**: 用户在侧边栏“新增事项”框输入一段话。
*   **测试案例**:
    - [ ] 输入 `今晚8点去超市买牛奶` -> 识别为今日日期，时间 20:00。
    - [ ] 输入 `明天下午3点接孩子` -> 识别为明日日期。
    - [ ] 输入 `下周五看牙医` -> 识别为下周对应的具体日期。
    - [ ] 输入 `4月3日去旅游` -> 识别为 2026-04-03（以后待办）。

### 2.2 循环意图提取
*   **场景**: 添加长期重复性事务。
*   **测试案例**:
    - [ ] 输入 `每周二购买 Game Night 零食` -> 提取循环模式：`Tuesday`。
    - [ ] 输入 `每天早上8点准时吃药` -> 提取循环模式：`Everyday`。
    - [ ] 输入 `每个周末打扫卫生` -> 提取循环模式：`Weekend`。

---

## 📝 第三部分：待办事宜管理 (Task Management)

### 3.1 列表自动分类
*   **场景**: 检查不同时间跨度的任务是否出现在正确区域。
*   **区域验证**:
    - [ ] **⚡ 今日急需处理**: 截止日期为今天及之前的任务。
    - [ ] **🌙 明日处理事项**: 截止日期正好为明天的任务。
    - [ ] **🗓️ 本周剩余任务**: 截止日期在明天之后到本周日之前的任务。
    - [ ] **⏳ 以后待办**: 所有本周以后（下周一及以后）的任务。

### 3.2 任务操作
*   **测试点**:
    - [ ] 点击任务左侧【复选框】-> 任务文字变灰带删除线 -> 刷新后该任务移动到“✅ 已完成事项”标签页。
    - [ ] 点击任务右侧【🗑️ 垃圾桶】图标 -> 任务是否从数据库和界面彻底消失？

---

## 🔄 第四部分：循环任务引擎 (Recurring Engine)

### 4.1 独立管理测试
*   **测试点**:
    - [ ] 进入“🔄 循环事项”标签页，是否列出了所有带循环标签的任务？
    - [ ] 在此页面删除一个循环任务，主页面的“影子提醒”是否同步消失？

### 4.2 智能影子注入 (Shadow Injection)
*   **场景**: 循环任务在每日清单中的提醒功能。
*   **测试点**:
    - [ ] 若有一个任务是“每周二...”，且今天正好是周二：
        - 该任务是否出现在“今日急需处理”中？
        - 图标是否为旋转的 🔄？是否没有复选框（仅作为提醒）？
    - [ ] 检查“明日处理事项”：如果明天符合循环条件，明日区域是否也出现了该任务的提醒？

---

## 📊 第五部分：数据归档与集成 (Integration)

### 5.1 已完成事项归档
*   **场景**: 历史任务回顾。
*   **测试点**:
    - [ ] 进入“✅ 已完成事项”标签页，所有已勾选任务是否按时间顺序排列？

### 5.2 家庭日历同步
*   **场景**: 侧边栏配置 Google Calendar。
*   **测试点**:
    - [ ] 在侧边栏修改 Email 地址 -> 点击“家庭日历”标签页 -> 日历内容是否自动切换为对应账号的 Embed 视图？

---

## 📱 第六部分：异常与响应式测试

### 6.1 异常处理
*   **测试点**:
    - [ ] 断开网络连接或 API Key 失效时，添加任务是否显示友好的提示而非崩溃？
    - [ ] 刷新网页时，背景渐变色和字体（Outfit）是否加载正常，无闪烁？

### 6.2 移动端匹配
*   **场景**: 使用手机/平板访问。
*   **测试点**:
    - [ ] 侧边栏在手机上是否能自动收起？
    - [ ] 任务列表在窄屏下是否由于 layout="wide" 而能自适应，且文字不会溢出？
