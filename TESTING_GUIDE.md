# 🏠 家庭事项管理系统 - 完整测试指南 (Testing Guide)

本项目是一个集成 AI 解析、循环任务引擎、持久化登录以及数据导出功能的家庭事务协作平台。本指南旨在详细列出所有最新功能测试点及应用场景，确保系统运行稳健。

---

## 🔐 第一部分：安全与访问控制 (Authentication)

### 1.1 基础登录测试
*   **场景**: 用户首次访问网站。
*   **测试点**:
    - [ ] 页面是否显示“家庭系统登录”标题？
    - [ ] 输入【错误密码】是否显示“🚫 密码错误”提示？
    - [ ] 输入【正确密码 (790228)】是否能瞬时跳转至主界面？

### 1.2 登录持久化 (Persistent Login)
*   **场景**: 用户已登录，希望刷新网页不掉线。
*   **测试点**:
    - [ ] 登录成功后，手动刷新浏览器页面，是否直接显示内容（无需再输密码）？
    - [ ] 关闭浏览器标签页，重新打开网址，是否依然保持登录状态？
    - [ ] 在侧边栏点击“🔴 退出登录”，刷新网页，是否要求重新输入密码？

---

## 🧠 第二部分：AI 智能任务解析 (AI Parsing)

### 2.1 自然语言日期提取
*   **场景**: 用户在侧边栏“新增事项”框输入一段话。
*   **测试案例**:
    - [ ] 输入 `今晚8点去超市买牛奶` -> 识别为今日日期，时间 20:00。
    - [ ] 输入 `明天下午3点接孩子` -> 识别为明日日期。
    - [ ] 输入 `下周五看牙医` -> 识别为下周对应的具体日期。
    - [ ] 输入 `4月3日去旅游` -> 识别为 2026-04-03（以后待办）。
    - [ ] **标题清理验证**: 检查任务标题中是否已自动剥离“明天”、“下午3点”等冗余词汇，仅保留核心任务内容。

### 2.2 循环意图提取
*   **场景**: 添加长期重复性事务。
*   **测试案例**:
    - [ ] 输入 `每周二购买 Game Night 零食` -> 提取循环模式：`Tuesday`。
    - [ ] 输入 `每天早上8点准时吃药` -> 提取循环模式：`Everyday`。
    - [ ] 输入 `每个周末打扫卫生` -> 提取循环模式：`Weekend`。

---

## 📝 第三部分：交互式任务管理 (Task Interaction)

### 3.1 任务新增与反馈
*   **场景**: 用户在侧边栏点击“立即添加”。
*   **测试点**:
    - [ ] **内容解析确认**: 点击后是否弹出“📋 事项添加结果”对话框？
    - [ ] **数据核对**: 对话框中显示的“内容”、“预计执行时间”和“循环模式”是否准确反映了 AI 的理解？
    - [ ] **自动复位**: 关闭弹窗（点击确定）后，侧边栏的输入框是否已自动变为空白？
    - [ ] **立即显示**: 任务是否已实时出现在右侧对应的列表区域？

### 3.2 列表分类与排序 (Updated!)
*   **区域与排序验证**:
    - [ ] **⚡ 今日急需处理**: 截止日期为今天及之前的任务。
    - [ ] **🌙 明日处理事项**: 截止日期正好为明天的任务。
    - [ ] **🗓️ 本周剩余任务**: 截止日期在明天之后到本周日之前的任务。
    - [ ] **⏳ 以后待办**: 所有本周以后（下周一及以后）的任务。
    - [ ] **🔥 顺序核对**: 检查以上所有区域内的任务是否**严格按时间先后顺序（Chronological Order）**排列？先要做（时间早）的事项必须排在上面。

### 3.3 任务状态操作 (Updated!)
*   **测试点**:
    - [ ] **行内编辑**: 点击任务右侧的 【✏️ 铅笔】图标，修改文字后保存，列表是否即时更新？
    - [ ] **单次完成 (常规)**: 勾选常规任务，任务文字变灰带删除线，移动至“✅ 已完成事项”。
    - [ ] **单次完成 (循环事项)**: 勾选今日清单中的循环影子任务（带 🔄 标志），该单次提醒应消失并移入已完成归档，但原循环规则保持不变。
    - [ ] **物理删除**: 点击任务右侧【🗑️ 垃圾桶】图标，任务彻底消失。

## 🔄 第四部分：循环任务引擎 (Recurring Engine)

### 4.1 核心规则管理
*   **测试点**:
    - [ ] 进入“🔄 循环事项”标签页，是否列出了所有带循环标签的任务模板？
    - [ ] **整体完成**: 在此页面勾选一个循环任务，代表该循环系列彻底结束，主页面的所有分身影子同步移动到已完成。
    - [ ] **删除规则**: 在此页面删除任务，主页面影子消失。

### 4.2 智能影子与单个勾选 (New!)
*   **场景**: 循环任务在每日清单中的个体化操作。
*   **测试点**:
    - [ ] 循环影子现已支持【复选框】。
    - [ ] 勾选今日的循环事项 -> 检查“已完成事项”标签页，是否出现类似 `任务名 (周期性于 YYYY-MM-DD)` 的存档？
    - [ ] 在已完成标签页取消勾选该记录 -> 检查该单次循环任务是否**重新变回待办状态**回到今日清单？

---

## 📊 第五部分：数据导出与存档 (Data Export)

### 5.1 任务报表导出 (Updated!)
*   **场景**: 用户希望离线保存所有任务。
*   **测试点**:
    - [ ] **按钮位置**: 确认按钮位于主页面顶部的“家庭事项管理中心”**主标题右侧**。
    - [ ] **下载触发**: 点击后是否成功下载 TXT 文件？
    - [ ] **文件命名**: 文件名是否包含日期时间（如 0311_1100）。
    - [ ] **内容核对**: 打开 TXT，检查表格是否整齐地列出了所有活跃任务、循环规则以及已完成的历史记录。

---

## 📱 第六部分：异常与响应式测试

### 6.1 效率与状态同步
*   **测试点**:
    - [ ] API 解析或数据库写入失败时，是否通过对话框弹出具体的失败原因？
    - [ ] 连续添加多个任务时，输入框是否每次都能正确清空且不报错？

### 6.2 跨设备适配
*   **测试点**:
    - [ ] 侧边栏在手机上是否能灵活收合？
    - [ ] 任务标题在窄屏下是否由于 layout="wide" 而自动换行，且不会遮挡操作按钮？
